■ 03 · THE HARD GATE — SUBSTANCE-LOCK ACULEUS — SALIENCE LAYER

Sentence-level binding is necessary — and not sufficient.

Every sentence can bind to a true receipt and the whole piece can still mislead — by selection, by omission, or by implication. So the verifier runs two tiers: one against each claim, one against the gestalt of the piece measured against the Read's bottom line.

TIER 1 — PER-CLAIM · BLOCK EVERY ASSERTIVE SENTENCE, INDIVIDUALLY
unbound_claim new_fact_introduced substance_drift attribution_mismatch thin_evidence_unflagged forbidden_move ethical_floor_violation
TIER 2 — GESTALT · BLOCK WHOLE PIECE VS THE READ'S BOTTOM LINE
misleading_implication selective_omissiona required_inclusion was dropped unsupported_causation
FAILURE HANDLING — BLOCK ISSUES ARE NEVER SILENTLY DROPPED
BLOCK ISSUE→ REPAIR LOOP → PASS 3
MAX 2 RETRIES→ FRONTIER-RENDERER FALLBACK→ HUMAN REVIEW
BLOCKING ISSUES ATTACHED
ALL OF IT UNDER A PER-MEDIA-RUN SPEND CAP.

■ 06 · THE BUILD — EIGHT MILESTONES ACULEUS — SALIENCE LAYER

The build — eight milestones, worked backwards.

Each milestone lands a deliverable, implements named runtime passes, and clears an explicit acceptance gate before the next begins.

MILESTONE DELIVERABLE IMPLEMENTS PASSES ACCEPTANCE GATE
M0 · CONTRACTS & SCHEMA Apply salience tables; type SalienceMap / RenderVariant / RenderBindings / RenderVerifierResult on the adapter. — Schema migrates from empty DB; adapter returns typed shells.
M1 · READ FREEZE + ATOMIZATION read_snapshot, claim_atoms, immutable AP-citation builder, eligibility gate. P0–P1 A finished Read yields a frozen, hashed atom + citation library; every atom binds to an evidence record; thin Reads rejected.
M2 · SALIENCE MAP PASS Salience-reasoner role + prompt + subset validator. P2 Per-demographic map validated; ZERO invented facts on a red-team set.
M3 · RENDER PASS Renderer role, four format specs, plain-language target, marker emission. P3 Draft per demographic × format with a binding marker on every assertion.
M4 · BINDING + SUBSTANCE-LOCK (HARD GATE) Binder, AP resolution, two-tier verifier, repair/escalation loop, spend cap. P4–P5 An injected unsupported claim is BLOCKED; a dropped counter-case is BLOCKED; the loop converges or escalates to human.
M5 · RENDER REVIEW UI Inspectable claims, cross-demographic diff, reviewer + customer gates. P6 Both approvals enforced; click-a-claim → receipt.
M6 · DELIVER + RECEIPTS APPENDIX Packaging, revalidation, citation appendix export. P7 Approved render exports with a fully resolvable citations file.
M7 · EVAL & TRAINING TWO tracks — INTEGRITY eval (gates) vs SALIENCE-QUALITY eval (informs). CROSS-CUTTING Substance-lock pass-rate + binding completeness gate promotion; renderer SLM runs shadow no-regression eval (ties to the existing TR-05 formatter gate). Salience-quality rubric informs but never overrides integrity.

■ 08 · MODEL BINDING ACULEUS — SALIENCE LAYER

Roles, not models. Bound through the run adapter.

ROLE PASS(ES) MODEL TIER WHY
DETERMINISTIC P0, P4 (BIND), P7 no model Hashing, freezing, injecting canonical citations — must be exact, not generated.
SALIENCE-REASONER P2 frontier Judgment about what's salient & the counter-case; highest reasoning load.
RENDERER P1, P3 Aculeus SLM
(frontier fallback)		 Fluent format-specific generation; the SLM is the trained target, frontier on repair escalation.
VERIFIER P5 frontier Adversarial gestalt judgment against the Read; integrity-critical.
■ RISK Over-constrained generation (substance + citation + ethical floor + coding + plain language + format) may exceed a small SLM; the frontier fallback and the M7 integrity eval exist to catch this.
■ 09 · DEFENSIBILITY ACULEUS — SALIENCE LAYER

Built to survive discovery, not just distribution.

WHAT THIS IS NOT
  • ✕Five sets of facts.
  • ✕Audience-tested fiction.
  • ✕Synthetic slop.
  • ✕Persuasion that can't survive discovery.
WHAT THIS IS
  • ✓One frozen Read per topic.
  • ✓Renderings vary in salience — never substance.
  • ✓The counter-case ships in every variant.
  • ✓A global ethical floor over per-demographic coding.
  • ✓The cross-demographic diff proves same-receipts-different-doors.
  • ✓Every claim keeps its path to the record.

“Source it, or don't say it — for every door in.”
